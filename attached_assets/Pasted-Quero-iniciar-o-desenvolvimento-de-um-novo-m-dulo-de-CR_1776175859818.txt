Quero iniciar o desenvolvimento de um novo módulo de CRM/Leads no ERP, mas este módulo deve nascer em modo de desenvolvimento/beta, sem afetar a operação atual e sem aparecer para os usuários comuns.

OBJETIVO GERAL
Criar uma estrutura inicial de CRM de captação de leads integrada ao ERP, preparada futuramente para n8n, Claude e automações, mas nesta etapa ainda sem ativar automações reais.

REGRA PRINCIPAL
Não alterar o fluxo atual do ERP.
Não quebrar telas existentes.
Não mostrar esse módulo no menu para vendedores, lojistas, financeiro ou admin comercial ainda.
Não liberar para operação real.
Criar tudo de forma isolada, testável e segura.

ACESSO
Nesta primeira fase, somente ADMIN_GERAL pode acessar e testar o módulo.

Criar uma rota interna/beta, por exemplo:
- /crm-leads-beta
ou
- /admin/crm-leads-beta

Essa rota pode aparecer apenas para ADMIN_GERAL ou ficar acessível por rota direta protegida.

NÃO exibir no menu principal para outros perfis.

FASE 1 — BANCO DE DADOS
Criar estrutura para leads comerciais.

Campos do Lead:
- id
- nome
- telefone
- email
- origem: META, GOOGLE, SITE, WHATSAPP, INDICACAO, OUTRO
- campanha
- interesse: MOTO, PECA, SERVICO, CURSO, OUTRO
- lojaId
- vendedorId
- status: NOVO, EM_ATENDIMENTO, PROPOSTA_ENVIADA, GANHO, PERDIDO, SEM_RESPOSTA
- prioridade: BAIXA, MEDIA, ALTA
- resumo
- proximaAcao
- dataProximoFollowUp
- observacoes
- createdAt
- updatedAt

Criar também histórico/interações do lead:
- id
- leadId
- usuarioId
- tipo: LIGACAO, WHATSAPP, EMAIL, OBSERVACAO, FOLLOW_UP
- descricao
- createdAt

FASE 2 — BACKEND/API
Criar endpoints protegidos para ADMIN_GERAL testar:

- POST /api/crm-leads
criar lead

- GET /api/crm-leads
listar leads

- GET /api/crm-leads/:id
detalhar lead

- PATCH /api/crm-leads/:id
editar lead/status/próxima ação

- POST /api/crm-leads/:id/interacoes
registrar interação

- GET /api/crm-leads-dashboard
retornar métricas para dashboard beta

Criar também um endpoint preparado para integração futura com n8n, mas ainda em modo teste:

- POST /api/integracoes/leads-test

Esse endpoint deve:
- receber lead externo de teste
- exigir token simples via header, exemplo: x-integration-token
- não disparar automações reais
- apenas salvar o lead no CRM beta
- marcar origem como TESTE ou conforme payload recebido

Adicionar variável de ambiente opcional:
CRM_LEADS_BETA=true
INTEGRATION_TOKEN=token_de_teste

Se a variável CRM_LEADS_BETA não estiver ativa, o módulo pode ficar bloqueado.

FASE 3 — FRONTEND / TELA BETA
Criar uma tela simples e limpa para ADMIN_GERAL testar o CRM de leads.

Nome da tela:
CRM Leads Beta

A tela deve conter:
- cards de resumo
- tabela de leads
- botão criar lead manual
- filtros por status, origem, prioridade e período
- botão ver detalhes
- botão registrar interação
- botão alterar status
- campo próxima ação
- data de próximo follow-up

Não precisa ficar perfeito visualmente nesta etapa, mas deve seguir o padrão visual do ERP.

FASE 4 — DASHBOARD BETA DO ADMIN COMERCIAL
Criar uma prévia de dashboard comercial, ainda visível somente para ADMIN_GERAL.

Esse dashboard deve mostrar:
- total de leads
- leads por origem
- leads por status
- leads por loja
- leads por vendedor
- leads por prioridade
- taxa de conversão
- ganhos x perdidos
- volume por período
- ranking de vendedores por quantidade de leads
- ranking de vendedores por conversão

Importante:
Este dashboard ainda não deve aparecer para ADMIN_COMERCIAL.
Ele deve ser apenas uma prévia beta para o ADMIN_GERAL validar.
